# AI 知识库项目

## 项目目标
构建一个本地 AI 知识库，支持链接抓取、自动整理、智能搜索、关联推荐。

## ⚠️ 上下文恢复规则（必须遵守）

**每次重启会话时**：
1. 先读 `STATUS.md` 了解项目进度
2. 再读 `AI知识库-产品设计文档.md` 了解产品设计
3. 然后继续工作

**每次完成重要变更后**：
1. 更新 `STATUS.md` 的「最后更新」时间
2. 更新「实现进度」章节
3. 更新「已知问题」章节（如有）
4. 更新「下一步行动」章节

## 使用的 Skills
本项目使用 superpowers 插件提供的以下 skills：
- **writing-plans**: 制定实施计划
- **executing-plans**: 执行计划
- **test-driven-development**: 测试驱动开发
- **systematic-debugging**: 系统化调试
- **verification-before-completion**: 完成前验证

## 技术栈
- Python 3
- 纯 Markdown 文件存储
- Claude API（语义处理）
- 零数据库、零向量库、零服务器

## 目录结构
```
├── CLAUDE.md           # 本文件
├── ARCHITECTURE.md     # 架构文档
├── raw/                # 原料层：原文、PDF（只读）
├── notes/              # 笔记层：AI 写的带判断的笔记
├── wiki/               # 编译层：主题页、实体页
├── briefs/             # 选题层：选题清单
└── scripts/            # 脚本：索引、关联、搜索
```

## Ingest 规则（语义活，交给 AI）

收到链接时：
1. 打开链接，读完全文
2. 提炼 2-3 句摘要 + 3-5 个要点
3. 打 3-5 个标签（不够就少打）
4. 存成 `notes/日期-标题.md`，格式如下：

```markdown
---
title: 文章标题
url: https://...
tags: [标签1, 标签2, 标签3]
date: YYYY-MM-DD
---

## 摘要
2-3 句话概括

## 要点
- 要点一
- 要点二
- 要点三
```

收到想法/灵感时同上，但不需要 url 字段。

## 工作原则
- **语义活交给 AI**：理解、判断、打标签、写摘要
- **机械活交给脚本**：搜索、索引、关联计算
- **先跑通最小版本**：功能能用就行，后面再优化
- **报错就甩回去**：不用看懂错误信息，直接复制给 AI 修

## 协调者模式（必须遵守）
主 agent 作为**协调者 + 审核者**，不独自吞下全部上下文和实现细节：
- **拆解任务**：把大任务拆成可独立完成的小任务
- **派发子 agent**：把调研、实现、测试任务交给子 agent 执行
- **审核结果**：检查子 agent 的输出质量
- **整合变更**：把多个子 agent 的成果合并
- **最终验证**：确保整体功能正常

禁止主 agent 一个人写完所有代码！
